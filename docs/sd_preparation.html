SD Card Preparation

1 Preparing the SD Card

Before deploying the PolliCam, the Raspberry Pi SD card must be flashed and configured with the core daemon software.

1.1 Flashing the OS

  1. Download the Raspberry Pi Imager.
  2. Select Raspberry Pi OS Lite (64-bit).
  3. Choose your target SD card.
  4. Click the gear icon to pre-configure:
    • Hostname (e.g., pollicam-01)
    • Enable SSH (Use public-key authentication if possible)
    • Set up your Wi-Fi credentials (for your mobile hotspot)
    • Set the locale settings.
  5. Write and verify the image.

Tip for Field Access: Because you set a unique Hostname (like pollicam-01), you don’t need to hunt for its IP address on your phone. You can connect using a stable address based on its ID! Just SSH into <hostname>.local (for example, ssh pi@pollicam-01.local).

1.2 Software Installation

Once booted, SSH into the Pi using the hostname you configured in the previous step.

# Connect to the Raspberry Pi
ssh pi@pollicam-01.local

Then, simply run this single installation command to automatically set up the camera, install all dependencies, and configure it to boot automatically:

curl -sSL https://raw.githubusercontent.com/jakubstenc/YOLO_camera/master/install.sh | bash

Once the installation finishes, the web interface will run automatically as a background service (pollicam-ui.service).

You can access the camera interface from your phone or computer at: https://<hostname>.local:5000

Warning: Because the camera generates a self-signed security certificate to allow GPS access, your browser will show a “Your connection is not private” warning. This is expected! Simply click Advanced and then Proceed to [hostname] (unsafe).

1.3 Optional: Predictable Android IPs

Android phones do not natively support .local hostnames when acting as a mobile hotspot. To easily bookmark the camera on an Android device without guessing its IP address, you can assign it a Predictable IP.

Run the following script to permanently lock the camera’s IP based on its hostname (e.g., pi1 becomes ...101, pi2 becomes ...102):

curl -sSL https://raw.githubusercontent.com/jakubstenc/YOLO_camera/master/lock_ip.sh | bash

Note: Your SSH connection will drop as soon as the IP changes. Reconnect using the new IP address.
